sign.html — Wizard de signature de bail à distance

Mockups statiques (zéro JS de logique). Chaque variante couvre tout le flux : consentement → lecture → signature → confirmation → succès, plus les 4 états d'erreur (lien expiré · déjà signé · pas votre tour · échec réseau). Comparez les 3 layouts aux 3 formats côte à côte. Page tierce, mobile-first (locataire sur téléphone).

A — Pleine page mono-colonneÉtapes linéaires, scroll vertical, une étape après l'autre. Simple, classique, lisible.
B — Lecture haut + barre collée basAction primaire et pad de signature dans une barre sticky en bas, façon app mobile. Pouce toujours servi.
C — Accordéon4 panneaux dépliables, un seul ouvert à la fois. Vue d'ensemble du parcours + progression visible.
ℹ️ Dans chaque fichier, tous les états sont empilés dans des <section> avec un bandeau « État : … ». Dans la variante C, l'accordéon du premier écran est réellement cliquable (CSS :target, sans JS). Données factices en dur.
