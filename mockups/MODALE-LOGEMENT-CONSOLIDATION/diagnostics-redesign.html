🏷 Dossier de Diagnostic Technique (DDT). Remplis une fois — l'app calcule l'expiration, prévient dans l'Agenda et signale les diagnostics expirés dans les actions prioritaires. Seuls les diagnostics applicables à ce logement sont affichés (selon le contexte ci-dessous). À joindre au bail à la signature (loi 89-462 art. 3-3, sous peine de nullité).
🔍 Non affichés (non applicables à ce logement) : Termites, Mérule, Bruit aérien (PEB). Coche la zone correspondante dans le contexte ci-dessous pour les activer.
⚙️ Contexte — détermine quels diagnostics sont obligatoires
Plomb si < 1949 · Amiante si < 1997 · Gaz/Élec si installation > 15 ans · ERP/termites/mérule/bruit selon zones. L'année de construction sera héritée de l'immeuble (phase B3).
Variante A — Cartes. Une carte par diagnostic, barre de couleur à gauche selon le statut. Le plus aéré et lisible, idéal mobile. Coût : plus haut verticalement (scroll) sur PC quand il y a 6+ diagnostics.
Variante B — Tableau clarifié. On garde le tableau (compact, vue d'ensemble immédiate sur PC) mais on clarifie : libellés en clair + sigle + bulle « ? », lignes colorées par statut, détail DPE en sous-ligne. Sur téléphone le tableau se transforme en cartes empilées. Coût : reste dense, moins respirant que A.
Variante C — Accordéon. Tout est replié par défaut : on ne voit que le libellé + le statut. On déplie un diagnostic pour le remplir. Réduit au maximum la sensation de « trop de champs d'un coup » (« fill once » guidé). Coût : un clic de plus pour voir/éditer chaque diagnostic.
Retiré : le bouton « 📎 Aperçu récap DDT » qui était sous le tableau — la synthèse part déjà dans le bail (page Annexes obligatoires).
🔄 DPE via ADEME — flux & pop-ups (switch « ADEME (DPE) » en haut)
Recherche : bouton « Récupérer depuis l'ADEME » sous la ligne DPE → panneau avec 2 modes : par adresse du logement (pré-remplie, sert de filtre) ou par n° DPE exact. Données officielles, gratuit, appel direct navigateur (CORS ouvert), mention « source ADEME ». Dans le mockup, « Rechercher » déclenche le pop-up « Données ≠ » pour la démo.
Pop-up « Adresse ≠ » : si l'adresse du DPE trouvé diffère de celle du logement dans l'app → on prévient (mauvais DPE ? erreur de n° ?) avant d'écraser quoi que ce soit. Choix : annuler / utiliser quand même.
Pop-up « Données ≠ » : si les données ADEME diffèrent du document (classe / conso…), tableau de comparaison + lien « Télécharger le dernier DPE (n° XXX) » vers l'Observatoire ADEME. Choix : garder mes valeurs / utiliser l'ADEME.
À confirmer (implémentation) : pour comparer « ADEME vs document », il faut connaître les valeurs du document — soit l'utilisateur les a saisies, soit on lit le PDF. Si on ne lit que le n° sur le PDF, la comparaison « Données ≠ » se base sur les valeurs déjà saisies dans le formulaire (sinon pas de comparaison : on remplit directement). À trancher quand on codera.
