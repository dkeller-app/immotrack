🗺 Schéma général — Source unique de vérité par champ

Chaque champ vit à UN SEUL endroit. Les autres entités lisent par référence via des helpers.

🏢 IMMEUBLE — le bâtiment (8-11 champs)

  • Adresse (rue, CP, ville)
  • Année construction
  • Période de construction
  • Régime juridique (copro / mono)
  • Syndic (si copro)
  • Équipements communs (ascenseur, gardien, interphone, digicode)
  • Valeur estimée, travaux, notes

🏠 BIEN (lot) — appartement / studio / garage (35-40 champs)

  • Réf, étage, n° apt
  • Type (T2, T3, garage…) + typeUsage
  • Surface, nb pièces, description
  • DPE {classe, GES, date, valeurs}
  • État des risques {ERP, plomb, amiante…}
  • Chauffage + ECS (internes au lot)
  • Équipements internes (cuisine, sanitaires, fibre/ADSL)
  • Mobilier (si meublé)
  • Lot copro, tantièmes, num fiscal
  • 📷 Photos, annonce
  • ⬆ Hérite de l'immeuble : adresse, période, régime juridique
↓ a 0..N baux dans le temps

📜 BAIL — le contrat juridique (30 champs)

  • Type (nu, meublé, étudiant, mobilité, garage, autre)
  • Locataires[] (parties)
  • Dates début / fin / IRL
  • Conditions : loyer HC, charges, DG, jpay
  • Zone tendue, encadrement, premiereLoc
  • Garant, plafond caution
  • Signatures + bailSnapshot (figé légalement)
  • Clôture {finEffective, motif, DG restitué}
  • ⬆ Lit le bien via _readLogForBail(bail, log)

👥 LOCATAIRE — la personne (8 champs)

  • Civilité + nom (+ prénom)
  • Naissance (date + lieu)
  • Contact : tél + email
  • Adresse précédente
  • Futur : profession, employeur, revenus (dossier locataire)
💡 Logique « se protéger » : chaque entité ne contient QUE ce qui la définit. Pas de double saisie, pas d'auto-incrimination par champ manquant (cf règle v14.8). Le bail (document signé) renvoie au logement et à l'EDL pour les détails contradictoires. L'inventaire mobilier vit dans l'EDL (v15.205), pas dans le bail.
🎨 Règle UX transverse — « choix prédéfini + ajout libre toujours »
Partout dans l'app où on propose des choix (checkboxes, listes, équipements, catégories…) : il DOIT y avoir une zone d'ajout manuel libre en complément. Le choix prédéfini est un kit de démarrage, pas une cage.

Exemples : équipements communs de l'immeuble (vue 3), chauffage (vue 4), 11 catégories décret 2015-981 dans l'EDL meublé (déjà livré v15.205), types de bail, motifs de fin de bail, catégories de mouvements bancaires…

Pattern UI : checkboxes/options + sous le bloc → input texte « + Autre (préciser) » + bouton « + Ajouter » → l'ajout devient un badge cliquable (✕ pour supprimer) accumulé en dessous.
📊 Champs supprimés / déplacés (récap)
❌ SUPPRIMÉS du log
  • hc, ch, dg, irl (loyer du bail courant)
  • debut, fin (dates du bail courant)
  • locataire, tel, mail (du bail courant)
  • adr (hérité de l'immeuble)
  • periodeConstr, regimeJuridique (idem)
→ Remplacés par getCurrentXxx(ref) + _logResolveAddress(log)
❌ SUPPRIMÉS du bail
  • adrBien, ftype, etage, surf, npp…
  • 11 booléens chauffXxx + chauff
  • 3 booléens ecsXxx + ecs
  • dpe, ges, dpeDate, erp, plomb, amiante…
  • equipCuisine, equipSanitaires, techInfo, depensesEnergie
→ Remplacés par _readLogForBail(bail, log)
