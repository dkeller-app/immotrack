Décisions spécifiques mobile

  • Topbar : thèmes inclus dans le topbar mobile (3 boutons compacts 22×20px à droite de la période)
  • Mode (Solo/Premium/Gestionnaire) caché en mobile (rare action, accessible via ⚙ Plus)
  • Bottom nav 4 onglets : Accueil (active) · Tableau de bord · Loyers · Plus (menu avec Logements/Locataires/Baux/Agenda/etc.)
  • QA tiles compactes : sous-titre raccourci (ex "16 lots · 1 vac" au lieu de "16 lots · 1 vacant")
  • Donut compact 100×100 avec breakdown stacked à droite
  • À traiter : grid 2 cols + titre raccourci "Impayé · 460€" au lieu de "Impayé Mai · 460 €"

Décisions spécifiques tablette

  • Sidebar comme sur le mockup tablet précédent (déjà validé)
  • Pas de bottom tabs (la sidebar suffit, plus naturel sur tablette)
  • Topbar : tout visible (thèmes + mode + période + drive) — la place le permet
  • Portrait : row1 = QA 2x2 + Donut côte à côte (1fr 1fr)
  • Landscape : row1 = QA 2x2 + Donut côte à côte mais ratio 1.1fr 1fr (QA légèrement plus large)
  • À traiter : 3 cols en portrait, 4 cols en landscape

Si tu valides → impact code

  • Tout reste dans #p-accueil (HTML déjà existant)
  • Uniquement du CSS responsive à ajouter dans main.css :
    • @media (max-width: 767px) : mobile (bottom nav + grid 2 cols)
    • @media (min-width:768px) and (max-width:1023px) : tablet portrait (sidebar collapsed + grid 3 cols)
    • @media (min-width:1024px) and (max-width:1279px) : tablet landscape (sidebar full + grid 4 cols)
  • Bottom nav mobile : nouveau composant HTML (en bas de body), affiché seulement en mode mobile via CSS
  • Sidebar mobile : cachée par défaut (offcanvas via ☰), déjà géré dans le code existant pour mobile
  • Estimation : 1 session ~2h CSS responsive + 30min nouveau bottom nav HTML
