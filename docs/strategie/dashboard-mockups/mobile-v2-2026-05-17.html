📐 Décisions design — points à valider

Topbar (52px de hauteur, 1 ligne)

  • ☰ hamburger (ouvre sidebar overlay pour navigation onglets globaux)
  • Mai 2026 ▾ pill cliquable (ouvre bottom-sheet pour choisir année+mois en 2 colonnes)
  • ☁ Drive sync (vert = OK, orange = en cours, rouge = expiré)
  • Cachés : Mode Solo/Premium/Gestionnaire + Thème ☀🎨🌙 → accessibles via Paramètres uniquement (changement rare)

Bottom nav (60px, fixe, padding-bottom: 18px pour safe-area iOS)

  • 4 onglets cloisonnés (one-screen-mobile principle, équivalent au "one-screen-PC")
  • 📊 Synthèse : hero actions + encaissement + 4 KPIs + projection (≈ 1 écran)
  • 💰 Cash : graphique 12 mois plein + 3 KPIs YTD empilés
  • 🎯 Sujets : liste verticale tous sujets (jusqu'à 20 items, scrollable)
  • 🏠 Entités : 1 card par entité avec ses immeubles + lots responsive 3 cols

Spécificités mobile vs desktop

  • Cash-flow KPIs : 3 lignes empilées au lieu de 3 colonnes (évite overflow des valeurs)
  • Sparkline cash-flow : rouge si net négatif (fix BUG-DASH-SPARK-COLOR backlog)
  • Entity lots : grid 3 colonnes fixe (au lieu de flex-wrap qui débordait)
  • Hero actions : max 2 visibles + bouton "DÉMARRER" pour drill-down complet
  • Sidebar entités épinglées : retirée du mobile (accessible via ☰)

Si tu valides → impact code

  • Nouvelle fonction _renderDashV4Mobile(ctx) activée si window.innerWidth ≤ 767px
  • Nouveaux composants : .v4m-bnav, .v4m-tab, .v4m-topbar, .v4m-ent-lots
  • CSS isolé sous body[data-dash-v4="on"][data-mobile="on"] pour ne rien casser desktop
  • Estimation : 1 session dédiée (~3-4h) — sandbox index-test.html d'abord
